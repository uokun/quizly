# コンテキスト情報の集約基盤 (_CONTEXT.md)

このファイルは、プロジェクト全体の文脈（Context）や基本的な設計方針、重要な取り決めをAIやユーザー間で共有・永続化するためのファイルです。

- **役割**: 一時的な作業ファイル（task.md, implementation_plan.md）で検討された重要な知見を、タスク終了後にここへ統合（Knowledge Sync）します。
- **管理方法**: サブドメインや特定の機能が増えた場合は、`docs/antigravity` ディレクトリ内に別ファイル（例: `feature_A_CONTEXT.md`）として分割・追加していくことが推奨されます。

## 現在の基本設計方針

1. **言語とコミュニケーション**:
   - 言語は一貫して日本語を用い、作業プロセスは `workflow` スキルの規則に従う。
2. **バージョニング戦略** (CHANGELOG.md に準拠):
   - 変更はまずパッチバージョン `0.0.X` として進める。
   - マイナーバージョン `0.X.0` は**ユーザーの明示的な承認**がある場合のみ引き上げる。
3. **UI/UX・フロントエンド**:
   - ダークモード/ライトモードの両対応を必須化。
   - レスポンシブデザイン（Mobile First）を厳守。
   - CSSは既存資産（変数・クラス）を再利用することから検討を始める。
